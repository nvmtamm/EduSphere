# Testing

## Testing Strategy
Testing should cover the platform at unit, integration, and workflow levels.

## Test Layers
- Unit tests for business rules and helpers
- Integration tests for persistence and API behavior
- End-to-end tests for learner and admin journeys

## Coverage Priorities
- Authentication and authorization
- Course enrollment and progress updates
- Content publishing and visibility
- Reporting and administrative actions

## Quality Expectations
- Tests should be deterministic and repeatable
- Critical paths should have regression coverage
- Failed tests should point to a specific layer and cause
